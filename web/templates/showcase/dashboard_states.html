Dashboard states

The dashboard is a three-state surface: zero (no projects exist — workspace is fresh or fully cleaned up), idle (projects exist, no running timer), running (a timer is currently running). State is computed in tracking.dashboardStateFor and passed to the template so each state renders a cohesive layout rather than three empty cards competing for attention.

Spec: openspec/specs/ui-partials/spec.md (Dashboard surface renders three documented states).
